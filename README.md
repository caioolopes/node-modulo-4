# node-modulo-4

Neste módulo, foram aplicados os fundamentos de Domain-Driven Design (DDD) e Clean Architecture no desenvolvimento de uma API REST para um fórum.
A arquitetura foi estruturada com base em boas práticas de separação de responsabilidades, incluindo a implementação de Value Objects, modelagem de relacionamentos, tratamento funcional de erros, desenvolvimento de casos de uso e testes unitários.
Também foram explorados conceitos de Subdomínios e Domain Events.
